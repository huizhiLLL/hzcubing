# 项目现状

更新时间：2026-06-17

## 项目定位

HZCubing 是面向魔方玩家的社区成绩系统，核心围绕用户、成绩、榜单、个人主页和活动项目展开。

当前项目采用前后端分离方式维护：

- 前端：`src/`，Vue 3 单页应用，负责页面、交互、状态管理和 API 调用。
- 后端：`server/`，Express + MongoDB API 服务，负责鉴权、用户、成绩、榜单、首页统计和整活项目数据。
- 静态资源：`public/`。
- 项目文档：`docs/`，按 `project.md`、`architecture.md`、`roadmap.md`、`api.md` 四份文档维护。

## 当前可用能力

### 前端

已完成的主要页面与能力：

- 首页：项目入口、后端汇总统计、项目展示。
- 登录注册页：支持登录、注册、密码可见切换、注册密码一致性提示和登录后跳转。
- 榜单页：支持按项目查看单次和平均榜单，支持后端搜索、移动端卡片展示、高亮当前用户和定位到我。
- GR / 纪录页：展示当前各项目最佳纪录与近期破纪录信息。
- 选手总览页：支持搜索、排序、分页和错误 / 空数据状态区分。
- 个人主页：展示资料、个人最佳、最近成绩、创建的整活项目，并提供管理成绩入口。
- 成绩提交页：支持按分类二级选择项目并提交个人成绩，通过独立管理区编辑、删除个人成绩。
- 设置页：支持头像、昵称、WCA ID、简介等资料维护。
- 整活项目管理页：管理员可维护动态整活项目。

当前前端已经形成的共享层：

- `src/api/index.js`：统一 axios 实例、鉴权 token 注入和 API 封装。
- `src/stores/user.js`：用户登录态、当前用户信息和本地持久化。
- `src/stores/records.js`：成绩加载、首页统计、后端榜单、GR、个人最佳和成绩写入能力。
- `src/stores/events.js`：官方项目、趣味项目、动态整活项目的合并与查询。
- `src/stores/theme.js`：主题状态。
- `src/stores/toast.js`：全局 toast 消息。
- `src/components/common/`：按钮、图标按钮、分段控件、确认弹窗、状态块、页面头、卡片等公共组件。
- `src/utils/recordRanking.js`：基于已加载成绩计算个人最佳排名映射的兼容辅助逻辑。

### 后端

已完成的主要接口域：

- `/api/auth`：注册、登录、获取当前用户、QQ 绑定、QQ 反查、AstrBot 插件入口。
- `/api/users`：用户资料、用户列表、用户总览、用户详情。
- `/api/records`：成绩创建、查询、更新、删除、个人最佳、全站最佳、单项目榜单、GR 历程、近期破纪录。
- `/api/stats`：首页轻量汇总统计。
- `/api/meme-events`：整活项目公开查询和管理员 CRUD。
- `/health`：服务健康检查。

当前后端已具备的基础设施：

- Express 应用入口：`server/index.js`。
- MongoDB 连接：`server/config/database.js`。
- JWT 鉴权中间件：`server/middleware/auth.js`。
- 统一错误处理中间件：`server/middleware/errorHandler.js`。
- Mongoose 模型：`User`、`Record`、`MemeEvent`、`Counter`。

## 当前重点

当前项目已具备可用 MVP 主链路，并完成了一轮前端体验与后端聚合接口收口。现阶段重点不是扩大功能范围，而是让已有主链路更稳定、更一致、更容易继续迭代。

优先关注：

- 前后端职责边界保持清晰，聚合统计、榜单排序、搜索分页等全局一致能力优先由后端提供。
- 前端继续复用公共组件、store、工具函数，减少页面内重复逻辑。
- 后端逐步拆出 service 层，避免路由文件继续承载过多业务聚合。
- 对成绩排名、GR 计算、用户概览、首页统计、榜单聚合、整活项目变更等高风险逻辑补测试。
- 对 QQ / 外部插件入口补安全边界和调用治理。

## 暂不做的内容

除非有明确任务，不优先推进：

- 大规模 UI 重做或重新设计整套视觉系统。
- 切换前端框架、状态管理方案或构建工具。
- 切换后端框架、数据库或核心鉴权方案。
- 引入重量级组件库、ORM、任务队列或复杂微服务架构。
- 直接在远端服务器长期改代码。
